¿Cuál es la diferencia entre Internet y la World Wide Web?

Podemos definir a Internet como la red de computadoras conectadas en las que funciona la web, a través de estas viajan los archivos que usamos diariamente.

Por su parte la World Wibe Web (también conocida como WWW) es un sistema interconectado de páginas web públicas que son accesibles a través de Internet.

¿Cuáles son las partes de un URL?

Una URL cuenta con diferentes partes, las cuales son:

  • El Protocolo: En su mayoría HTTP, sirve para transferir información entre diferentes actores en Internet
  • El subdominio: Es todo lo que aparece entre el protocolo y el primer punto que separa al dominio.
  • El Dominio: Nombre único que sirve para identificar una página web
  • Ruta: Indica páginas y subpáginas que podemos encontrar en un sitio web.
  • Parámetros: Datos adicionales utilizados para poder hacer consultas.t

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Se emplea para solicitar datos de un recurso específico, los datos no se modifican.
HEAD: Se solicitan datos solo del encabezado, no se toma en cuenta el cuerpo del archivo
POST: Se emplea para poder enviar datos a un servidor y así crear un recurso.
PUT: Se emplea para actualizar el recurso existente en un servidor.
PATCH: Es utilizado para poder realizar modificaciones parciales a un recurso.
DELETE: Se emplea para poder eliminar un recurso.

Debido a que en esta consulta se pretende hacer envió de información y tomando en cuenta que es un primer registro para poder ingresar a la página web, se debería de usar el método POST.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Este código generalmente es usado en respuesta a un servidor HTTP y significa que la petición fue correcta y se respondió sin problemas.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Debería ser responsabilidad del desarrollador corregir esto si es que el enlace por el que intentó acceder el usuario es un enlace que el mismo sitio web proporciona, ya que nos estaría dando a entender que el enlace está roto.

De igual forma debería de haber una página personalizada para el error 404, esto con la finalidad de no romper con la estética del sitio

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Cuando se genera un código de error 500 significa que el servidor encontró una condición inesperada que le impide completar la petición.

En este caso si es la responsabilidad del desarrollador corregir el problema, ya que significa que existe un problema que no le deja al sitio conectarse al servidor.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Cuando existe un atributo desaprobado quiere decir que es un atributo que se considera obsoleto esto debido a que hay nuevas y mejores estructuras que pueden ser usadas en su lugar- Algunos ejemplos son:

  • menu: Reemplazado por el tag HTML ul.
  • dir: Atributo remplazado por el tag HTML ul
  • applet: Reemplazado por el tag HTML object.
  • acronym:Para abreviaciones se mantiene la etiqueta abbr.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Cuando se empleaba HTML4 se necesitaban tecnologías como flash o actionscript para poder trabajar con vectores, gráficos o imágenes.

Por su parte HTML5 es capaz de manejar todos estos recuros debido a que el mismo estándar lo permite, de igual forma viene con soporte para audio y video.

¿Qué componentes de estructura y estilo tiene una tabla?

  • Para modificar la leyenda: caption
  • Para modificar el encabezado: thead
  • Para modificar el cuerpo de la tabla: tbody
  • Para modificar las filas: tr
  • Para modificar las celdad: td, th

¿Cuáles son los principales controles de una forma HTML5?

Existen diferentes tipos de entrada como lo pueden ser: botones, campos de texto, casillas de verificación, casillas para ingresar contraseñas, imágenes, radios, submit y text.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Debería ser una imagen que no presentará colores que pudieran hacer que el texto de nuestra página se pierda, de igual forma debería ser una imagen que no sature los demás elementos del sitio web.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de una propiedad de estilo entre %, px y pt?

Dependerá mucho de lo que estemos haciendo o de lo que queramos realizar, por ejemplo, se pueden usar la propiedad de estilo pt si tenemos pensado que nuestro diseño pueda ser impreso, estas unidades son de tamaño fijo y no pueden escalarse en tamaño.

Cuando usamos porcentaje (%)el texto que estemos usando será completamente escalable, esto es beneficioso si se accede a nuestro sitio a través de dispositivos móviles.

Por último, los píxeles son unidades de tamaño fijo que se utilizan en los medios de pantalla, como la pantalla de la computadora. Presenta una buena representación a través de navegadores, sin embargo no es escalable para móviles.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Siempre será mejor poder contar con una versión minimizada de un archivo, esto permite que el tamaño del archivo sea más ligero y por ende, la carga del mismo será más rápida, permitiendo así que el sitio cargue y responda con una mayor velocidad.
